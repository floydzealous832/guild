# Expected output

Captured snapshots from a reference run of this example. Generated by running
`./setup.sh` then `./run.sh` in the sandbox, then querying:

- `guild quest list -p guild-example-01-hello-guild` → `quest-list.txt`
- `guild lore list -p guild-example-01-hello-guild` → `lore-catalog.txt`
- `guild quest bounties -p guild-example-01-hello-guild` (second call — after the inscribe and the quest post) → `session-start-output.txt`

The LORE and QUEST IDs shown in the snapshots are from the reference run and
will differ in yours — guild assigns IDs globally across all projects in the
database. The content and count (1 principle, 1 quest) are what should match.

`session-start-output.txt` is the capture that demonstrates guild's dual
feedback loop: the oath wall carries the principle AND the top bounty
surfaces the audit quest — both from a single `session_start` call.
